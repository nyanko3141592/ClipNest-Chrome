ClipNest — Privacy Policy

Last updated: March 30, 2026

Overview

ClipNest is a clipboard history and snippet manager for Chrome. It is designed with privacy as a core principle — all data stays on your device and is never transmitted externally.

Data Storage

ClipNest stores the following data locally in your browser using chrome.storage.local:

  • Clipboard history — text you have copied, stored temporarily for quick re-access
  • Snippets — text you have saved manually as reusable snippets
  • Settings — your preferences (keyboard shortcut, history size, auto-paste toggle)

Data Collection

ClipNest does not collect, transmit, or share any user data. Specifically:

  • No data is sent to any external server
  • No analytics or tracking of any kind
  • No user accounts or authentication
  • No third-party services or SDKs
  • No remote code execution

Permissions

  • storage — to save clipboard history, snippets, and settings locally in the browser
  • activeTab — to show the inline paste panel on the currently active tab when triggered by the user
  • scripting — to inject the content script that enables the inline panel and paste functionality on pages where it is not pre-loaded
  • contextMenus — to add ClipNest items to the right-click menu for quick access to history and snippets
  • Host permissions (all_urls) — the content script runs on all pages so the inline paste panel and clipboard chip are available wherever the user types

Data Deletion

You can delete all stored data at any time:

  • Clear clipboard history from the right-click menu or the management page
  • Delete individual snippets or folders from the management page
  • Uninstalling the extension removes all stored data
